Project "T.A.L.O.N."

A Live-Streaming, AI-Powered FPV Rover

Welcome to the project showcase for T.A.L.O.N. (Tactical Animal-Locating Optical Navigator). This project is a smart, telerobotic rover that combines remote operation with real-time, AI-powered object detection.

This application will guide you through the project's mission, design, hardware, software, and future potential. Use the navigation on the left to explore any section.

The Mission

"Can we build a low-cost, smart rover that does more than just drive?"

The Problem

  • Most simple robots are "dumb"â€”they can't see or understand their environment.
  • Their capabilities are limited to basic tasks like following a line or avoiding a wall.

The Goal

  • Create an intelligent, remote-controlled (telerobotic) system.
  • The robot must stream live video to an operator.
  • A separate AI "brain" must watch this stream and identify specific, pre-determined targets in real-time.

System Architecture

This is a "co-op" system with three main parts that talk over a Wi-Fi network hosted by the rover itself. Click on any component to learn more.

The Rover (ESP32-CAM)

  • Acts as the central hub of the entire system.
  • Hosts its own Wi-Fi Access Point (SSID: `Robot_FPV_Control`).
  • Streams live video from its camera via a web server.
  • Listens for and executes motor commands sent over HTTP.

The Operator (Your Phone)

  • Connects directly to the robot's Wi-Fi network.
  • Loads a webpage served by the ESP32 (at `192.168.4.1`).
  • This webpage contains a virtual joystick for real-time control.
  • Sends `Forward`, `Left`, `Right`, and `Stop` commands to the rover.

The AI Brain (Your Laptop)

  • Also connects to the robot's Wi-Fi network.
  • Pulls the *same* live video stream that the phone would see.
  • Runs a Python script using OpenCV, cvlib, and TensorFlow.
  • Analyzes the video feed in real-time to find specified targets (e.g., "cat").
  • Displays the video with bounding boxes and "TARGET ACQUIRED" text.

Hardware Components

A simple build using low-cost, high-power components. The "secret" to success is the two-part power system to prevent stability issues.

Brain & Vision

ESP32-CAM: An all-in-one camera, CPU, and Wi-Fi module.

Chassis

A standard 2WD Robot Chassis with two motors and a caster wheel.

Motor Control

L298N Motor Driver to interface between the ESP32 and the motors.

Rover Power

1x 5V USB Power Bank for the power-hungry ESP32-CAM.

Motor Power

1x 4xAA Battery Pack (or LiPo) for the motors via the L298N.

The Software Stack

Two main codebases work together to bring the robot to life.

1. The Rover Code

(Arduino / C++) - `esp32_cam_robot.ino`

  • Uses `WebServer.h` to host the HTML/CSS/JS for the virtual joystick.
  • Uses `esp_camera.h` to initialize and stream the live video.
  • Listens for HTTP GET requests (e.g., `/control?cmd=forward`) to drive the motors.

2. The AI Brain

(Python) - `animal_finder.py`

  • Uses `urllib` to tap into the camera's `/stream` URL.
  • Uses `cvlib` and `tensorflow` to run a pre-trained `yolov4-tiny` model.
  • Prompts the user for a target (`cat`, `dog`, `bird`).
  • Uses `OpenCV` to draw bounding boxes and "TARGET ACQUIRED" text on the video feed.

Key Challenges & Solutions

Every project has its hurdles. Here's how we solved the main ones.

Power Stability

Problem: The ESP32-CAM reboots constantly if the power is unstable (a "brownout"), especially when Wi-Fi and the camera are active.

Solution: Using two separate, dedicated power sources: a 5V USB power bank for the ESP32 and a separate battery pack for the motors.

Video Stream Lag

Problem: The live video stream can have a 1-2 second delay, making the robot very difficult to drive in real-time.

Solution: Forcing the camera to a low-resolution `FRAMESIZE_QVGA` (320x240) in the Arduino code. This significantly reduces bandwidth and lag, providing a much higher frame rate.

Python Dependencies

Problem: `tensorflow` is a large and complex library with many dependencies that can be difficult to install correctly.

Solution: Carefully ensuring all libraries (`cvlib`, `opencv`, `tensorflow`) were installed and compatible with the specific Python version (3.11).

Live Demo Plan

This is how the full "co-op" system works in practice.

  1. Power On Robot: The ESP32-CAM boots up and creates its Wi-Fi hotspot.
  2. Start AI Brain: Run the Python script on the laptop. It will connect to the robot's Wi-Fi and ask for a target (e.g., "cat").
  3. Connect Operator: Connect the phone to the robot's Wi-Fi.
  4. Show AI Feed: The laptop screen now shows the robot's first-person view, with the AI actively scanning for the target.
  5. Drive Robot: Use the phone's virtual joystick to drive the robot around.
  6. Find Target: Point the robot at a picture of a cat. The AI feed on the laptop will light up with a "TARGET ACQUIRED" message.

Future Improvements

This project is a great platform. Here are the logical next steps.

  • ✔

    Full Autonomy

    Have the Python script send an HTTP command back to the robot (e.g., `/stop` or `/celebrate`) the moment the target is found.

  • ✔

    On-Board AI

    Use a more powerful board (like a Raspberry Pi or ESP32-S3) to run the AI model *on the robot itself*, removing the need for the laptop.

  • ✔

    "Search and Rescue" Mode

    Add a servo and gripper. When the target is found, the robot can autonomously approach and "rescue" it.

  • ✔

    Acoustic Feedback

    Add a buzzer to the robot that beeps when the AI finds the target, giving the operator an audio cue.
